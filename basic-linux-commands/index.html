Basic Linux Commands

on 25 March 2019

In this blog post, we will go through the list of must-know Linux commands like moving around your files, handling tasks, and tweaking settings. Once you've got those basic down, you'll be ready for more cool Linux stuff. Let's jump in ðŸš€

Files and navigating
  • cd <dir> - change directory to <dir> (<dir> will be directory name)
    • cd - change to home directory
    • cd .. - change to parent directory of the current one
  • cp <file1> <file2> - copy file1 to file2
  • cat <file> - output contents of file
  • ls - directory listing (list all files and folders on the current directory)
    • ls -l - formatted listing
    • ls -la - formatted listing including hidden filess
  • mkdir <dir> - create a directory <dir>
    • mkdir -p <dir> - create any non-existing directories as necessary
  • mv <file1> <file2> - rename file1 to file2
  • pwd - show current working directory (aka print working directory)
  • rm <file> - remove file
    • rm -f <file> - force remove file
    • rm -rf <dir> - force remove directory
  • touch <file> - create if not-existing or update modification and access time
  • tail -f <file> - output contents of file as it grows
System info
  • date - show current date time
  • uptime - show how long system has been running
  • whoami - who you are logged in as
  • w - display who is logged in and what they are doing
  • cat /proc/cpuinfo - display cpu info
  • cat /proc/meminfo - display memory info
  • free - show memory and swap usage
  • du - show disk usage
  • df - display free disk space
  • uname -a - show kernel configuration
Processes
  • ps - display currently active processes
    • ps aux - detailed output of acive processes
  • kill <pid> - kill a process with process id (<pid>)
Networking
  • ping <host> - send a ping message to <host>
  • whois <domain> - retrieve more specific detail for <domain>
  • dig <domain> - get DNS (Domain Name System) for domain
  • wget <file> - download a file
    • wget -c <file> - continue stopped download
  • curl <url> - output the webpage from an <url>
    • curl -o <output> <url> - output the webpage from an <url> to an <output> file
  • ssh user@host - connect to host as user
    • ssh -p port user@host - connect using port
Others
  • grep <pattern> <file> - search in <file> for <pattern>
    • grep -r <pattern> <dir> - search for <pattern> recursively in <dir>
  • whereis <app> - show possible locations of <app>
